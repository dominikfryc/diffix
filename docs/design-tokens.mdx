# Design tokens

Design tokens are the visual design atoms of the design system â€” specifically, they are named entities that store visual design attributes. We use them in place of hard-coded values (such as hex values for color or pixel values for spacing) in order to maintain a scalable and consistent visual system for UI development.

CSS custom properties (also known as CSS variables) are used for design tokens. CSS custom properties are a part of the CSS specification that allows developers to define variables in CSS. They are accessible using the `var()` function.

This project is based on [Diffix Design System](https://www.figma.com/file/oY7h7B44pQgwYROqtEVQrw/Diffix-%E2%80%93-Design-System?type=design&node-id=0%3A1&mode=design&t=nEm6cABKSQQVJtUa-1). You can customize it to match your brand using design tokens, see [Customization](/?path=/docs/customization--documentation).

## Color

Color is a key part of any design system. It helps communicate the brand, establish hierarchy, and create a sense of unity.

### Primitives

Design system is built on a foundation of color primitives. These are the core colors that are used throughout the system. Primitives should be only used for in theme definition and should not be used directly in components.

Diffix Design System is inspired by selected colors from [Tailwind CSS](https://tailwindcss.com/docs/customizing-colors).

### Aliases

Aliases are used to provide semantic meaning to colors. They are used to communicate the intent of the color, rather than the specific color itself. Aliases should be used in components instead of primitives. When creating a custom theme, aliases should be modified by changing the primitive they reference.
